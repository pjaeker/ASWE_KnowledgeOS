# Reference: Session Contract

## Dual-status model

A robust closeout must distinguish:

- `latest status update found`: the freshest repo status artifact supported by GitHub-first evidence
- `status referenced by entry`: the status document currently referenced by the canonical entry

These may differ. When they differ, report both and explain the operational consequence instead of forcing a false single status.

## Required anchor set

- canonical entry after closeout
- latest status update found
- status referenced by entry
- active workstream
- handoff status
- additional anchors
