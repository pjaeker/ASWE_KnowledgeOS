## Output template

1. Work mode
   - mapping-only / repo-evidence-only / mixed

2. Session-anchor alignment
   - inherited entry or equivalent anchor
   - inherited status or dual-status view
   - active technical workstream
   - drift or conflict: yes/no

3. Active scope
   - current slice, file family, or subsystem under inspection

4. Key findings
   - concise factual mapping conclusions only

5. Evidence anchors
   - files, extracts, overlays, or symbols that support the mapping

6. Confidence
   - low / medium / high with one-sentence reason

7. Open questions
   - only the gaps that affect the next slice decision

8. Recommended next action
   - smallest safe next thin slice or next read

9. Role handoff / next bounded role
   - none / aswe-test-agent / aswe-observer / aswe-triager / aswe-doc-agent

10. Likely files / paths
11. Anchor files to read first
12. Hidden coupling / risks
