## Assumptions

This skill is a bounded documentation wording role optimized for GPT-Pro as main agent.

- The doc agent inherits the effective session contract before suggesting file targets or wording.
- It works only from confirmed evidence and should not compensate for missing technical clarity with fluent prose.
- Repo-local `.codex` implementations may specialize the role, but they must preserve the no-second-writer-path rule.
- In package-local refactor work, the doc agent is still valid because wording and structure implications can be prepared without repo write access.
