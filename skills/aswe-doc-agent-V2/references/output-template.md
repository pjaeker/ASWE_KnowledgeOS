## Output template

1. Work mode
   - wording-only / repo-evidence-only / mixed

2. Session-anchor alignment
   - inherited entry or equivalent anchor
   - inherited status or dual-status view
   - active technical workstream
   - drift or conflict: yes/no

3. Active scope
   - target docs, file family, or wording surface under inspection

4. Key findings
   - concise wording-relevant conclusions only

5. Evidence anchors
   - confirmed findings, file candidates, or terminology anchors that justify the wording

6. Confidence
   - low / medium / high with one-sentence reason

7. Open questions
   - wording blockers or technical gaps that still matter

8. Recommended next action
   - smallest safe documentation follow-through step

9. Role handoff / next bounded role
   - none / aswe-mapper / aswe-test-agent / aswe-observer / aswe-triager

10. Target files that likely need wording changes
11. Concise proposed wording blocks
12. Frontmatter / structure implications
13. Evidence or terminology to preserve
14. Documentation risks or ambiguities
