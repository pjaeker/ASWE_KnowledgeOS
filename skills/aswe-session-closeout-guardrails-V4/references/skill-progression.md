# Reference: Skill Progression

- V1: baseline closeout skill with canonical closeout order, entry refresh last, handoff decision, and next-start packet
- V2: fresh-evidence preflight, dual-status reporting, GitHub-first source priority, and anti-stale-anchor guardrails
- V3: explicit closeout writingslice decisions for status, workstream, structural docs, entry, and handoff, plus required no-change justifications when no artifact should be written
- V4: optional, evidence-based `Recommended bounded roles for next slice` block, aligned with repo-wide role binding and without weakening one-writer-many-readers governance
