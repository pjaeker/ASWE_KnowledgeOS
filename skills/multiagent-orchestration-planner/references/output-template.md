# Default output template

## 1. Routed task
- object domain
- knowledge level
- abstraction level
- first access path
- wrong route avoided

## 2. Orchestration decision
- single-agent / bounded multi-agent / deeper orchestration
- why this is the smallest safe structure

## 3. Work structure
For each workstream or slice:
- purpose
- scope
- output artifact
- dependencies
- out of scope

## 4. Capability matrix
For each capability:
- required now / later / unnecessary
- why

## 5. Role mapping
For each role:
- capability covered
- input contract
- output contract
- non-goals
- handoff target
- readiness state

## 6. Information flow
Show the flow as a readable chain or small graph description:
- source input
- role or stage
- transformation
- output artifact
- next receiver
- stop or return condition

## 7. Authority and safety boundaries
- final authority
- write boundary
- verification point
- sensitive operations or permissions boundary

## 8. Risks and gaps
- likely failure modes
- missing roles or tooling
- assumptions that could break the plan

## 9. Smallest next step
- the next concrete action that would make the plan executable
