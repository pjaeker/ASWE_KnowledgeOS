# Output profile: repo-local architecture and governance tasks

Use this profile when the routed task primarily concerns a repository, architecture, governance, contracts, gates, or workstream structure.

## Additional planning emphasis
Prioritize:
- repo-local source of truth
- architectural boundaries
- policy, gate, and contract surfaces
- workstream decomposition
- promotion paths
- drift risks between reference, skills, runtime mirrors, and generated artifacts

## Additional capabilities to consider
- scope and SSOT mapping
- semantic consistency checking
- architecture synthesis
- gate and verify design
- documentation codification
- security and permission boundary review

## Suggested extra output sections
- affected SSOT anchors
- architecture boundary decisions
- gate and contract touchpoints
- workstream and artifact recommendations
- mirror-drift and second-writer-path risks

## Warnings
Do not treat a repo-local architecture task as a pure implementation task unless the routing step clearly places it there.
