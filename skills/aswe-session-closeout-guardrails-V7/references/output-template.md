# Output template

## Closeout checkpoint

1. session summary
2. leading repo evidence used
3. canonical entry after closeout
4. latest status update found
5. status referenced by entry
6. active workstream
7. handoff status
8. contract status
9. next recommended starting focus

## Artifact-family decisions

For each family, decide explicitly:
- RepoStatusUpdate: create / update / no-change
- Workstream: create / update / no-change
- Structural Reference / Protocol / Policy: create / update / no-change
- Codex-App Operating Guidance: create / update / no-change
- ENTRY_LATEST: refresh / no-change
- NextChat-Handoff: not-needed / optional-available / required-and-used

## Materialization block

If the closeout is actually materialized now, also report:
1. branch
2. commit
3. artifact families touched
4. files touched
5. next-start packet based on the resulting branch state
