## Output template

1. Observation mode
   - live-readonly / repo-evidence-only / mixed

2. Session-anchor alignment
   - inherited entry or equivalent anchor
   - inherited repo status or equivalent anchor
   - active technical workstream
   - drift or conflict: yes/no

3. Active scope
   - workspace / project / environment / service / deployment / host

4. Key findings
   - concise factual findings only

5. Evidence anchors
   - strongest evidence observed
   - latest runtime evidence used
   - whether evidence is live or repo-materialized

6. Ranked hypotheses and confidence
   - 1..N hypotheses ranked by support
   - confidence: low / medium / high

7. Open questions or missing evidence
   - unresolved gaps that materially limit confidence

8. Next safe diagnostic step
   - smallest read-only next step
   - if mutation is required, explicitly hand back instead of acting
