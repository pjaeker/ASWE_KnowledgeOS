## Output template

1. Work mode
   - verify-design-only / repo-evidence-only / mixed

2. Session-anchor alignment
   - inherited entry or equivalent anchor
   - inherited status or dual-status view
   - active technical workstream
   - drift or conflict: yes/no

3. Active scope
   - changed slice, verify surface, or command family under inspection

4. Key findings
   - concise verify-relevant conclusions only

5. Evidence anchors
   - files, workstream goals, overlays, or prior outputs that justify the checks

6. Confidence
   - low / medium / high with one-sentence reason

7. Open questions
   - missing facts that limit the verify plan

8. Recommended next action
   - smallest safe next verify or evidence step

9. Role handoff / next bounded role
   - none / aswe-observer / aswe-triager / aswe-doc-agent / aswe-mapper

10. Proposed verify / gates
11. Smoke commands or checks
12. Expected good / bad outcomes
13. Stop-and-ask triggers
